# CLAUDE.md

Это руководство для AI-ассистента (Claude Code) при работе с данным репозиторием.

## О проекте

Backend на **NestJS 11** (TypeScript 5.7), REST API для сайта (новости/посты, отделы, теги, клубы, события, книги, игровая система достижений и т.д.). Проект находится в процессе миграции данных со старой БД на новую схему Prisma.

## Стек технологий

- **Язык**: TypeScript (ES2023, strict null checks)
- **Фреймворк**: NestJS 11
- **ORM**: Prisma 6 (основная БД, MySQL) + TypeORM 0.3 (используется только для чтения старой БД при миграции данных)
- **Аутентификация**: Passport.js (JWT + Local стратегии), bcryptjs для хешей паролей
- **Файловое хранилище**: MinIO (S3-совместимое), обработка изображений — Sharp
- **Утилиты**: Luxon (даты), uuid, slugify (генерация slug из title)
- **Линтер/форматтер**: ESLint (typescript-eslint) + Prettier (singleQuote, trailingComma: all)
- **Тесты**: Jest (unit, *.spec.ts рядом с кодом) + Jest e2e (test/jest-e2e.json)

## Архитектура и соглашения

- Каждый домен — отдельный NestJS-модуль (module/controller/service/dto/entity).
- Слои: Controller (HTTP) → Service (бизнес-логика) → Prisma (доступ к БД).
- Валидация входных данных через DTO-классы.
- `PrismaModule` — глобальный модуль, доступен во всех модулях без явного импорта.
- Slug для сущностей (например, постов) генерируется автоматически через `slugify`.
- Стиль кода: Prettier (одинарные кавычки, trailing comma `all`), правило `@typescript-eslint/no-explicit-any` отключено, `no-floating-promises` — warning.

## База данных

- Основная БД — MySQL через Prisma (`DATABASE_URL`).
- TypeORM используется только в модуле `migration` для чтения старой БД (`SOURCE_DB_*`) и переноса данных в новую (`TARGET_DB_*`).

## Переменные окружения (.env)

- `DATABASE_URL` — строка подключения MySQL для Prisma
- `PORT` — порт сервера (по умолчанию 3333)
- `JWT_SECRET` — секрет для JWT
- `MINIO_*` — конфигурация MinIO (endpoint, port, user, pass, bucket)
- `SOURCE_DB_*` / `TARGET_DB_*` — параметры подключения для модуля миграции

## Правила работы ассистента

- **Язык общения и документации**: вся коммуникация с пользователем, комментарии в отчётах и любая создаваемая документация ведутся **на русском языке**.
- **Свобода редактирования кода**: разрешено добавлять и удалять любые строки кода в проекте без отдельного разрешения пользователя на каждое изменение.
- **Папка `temp/`**: рабочая директория ассистента с полным доступом — можно свободно создавать, изменять и удалять любые файлы внутри неё без подтверждения. Используется для временных файлов, черновиков, экспериментов.
- **Папка `reports/`**: после каждой завершённой задачи (сессии работы) ассистент обязан создавать новый файл-отчёт о проделанной работе на русском языке (например, `reports/YYYY-MM-DD-HHmm-краткое-описание.md`). Полный доступ к этой папке — без подтверждений.
- Несмотря на свободу редактирования кода, при потенциально опасных или необратимых операциях (force push, удаление веток, изменение БД в проде, `git reset --hard` и т.п.) — по-прежнему уточнять у пользователя.
